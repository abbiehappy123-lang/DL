# 自定义规则

用户单独定义的设计规则，优先级高于默认规范。

---

## ⛔ Token 防幻觉声明（最高优先级）

> **禁止依赖记忆输出任何 Token 值。** 以下规则在每次涉及颜色、间距、圆角、阴影时强制执行。

| # | 禁令 | 正确做法 |
|---|------|---------|
| 1 | 禁止凭记忆写颜色值 | 所有颜色必须从 `references/colors.md` 读取后使用 |
| 2 | 禁止凭记忆写间距/圆角值 | 所有间距与圆角必须从 `references/spacing.md` 读取后使用 |
| 3 | 禁止自造 CSS 变量名 | 只能使用规范文件中明确存在的变量名，不可推测拼写 |
| 4 | 禁止使用 `#000000` 纯黑 | 最深文字色为 `#222222`（`--text-primary`），无纯黑 |
| 5 | 禁止使用 `#FFFFFF` 纯白作为页面背景 | 页面底色为 `#F0F1F5`（`--surface-canvas`），卡片白为 `#FFFFFF`（`--surface-default`） |
| 6 | 颜色/Token 有疑问时 | 先 `cat references/colors.md`，确认后再使用，不猜测 |

## 页面布局规则

| # | 规则 | 说明 |
|---|------|------|
| 1 | Phone frame 宽度为 **375px**，高度 **844px** | 设计基准设备为 iPhone 14（逻辑尺寸），禁止使用 390px 物理尺寸 |
| 2 | 页面内容区左右间距为 **12px** | Content 区域的 paddingLeft / paddingRight 统一为 12px，确保卡片宽度 351px = 375 - 12×2 |
| 3 | 卡片宽度为 **351px** | 所有手动组合卡片容器的宽度固定为 351px，禁止使用 355px、358px 或其他值 |
| 4 | 卡片容器四边 padding 均为 **16px** | 所有卡片容器的四边内边距统一为 16px，不可为 0 |
| 5 | Header ↔ Content 间距 **8px**，Content ↔ Footer 间距 **16px** | itemSpacing 按层级设置，不可统一用同一个值 |
| 6 | 页面级卡片圆角为 **16px** | 直接放置在页面内容区的卡片容器圆角统一为 16px（`Radius/Large`） |
| 7 | 卡片内子卡片圆角为 **12px** | 嵌套在卡片内部的子卡片（如 CardContent subtle 模式下的内容区块）圆角为 12px（`Radius/Medium`） |

## Home Indicator 规则

iPhone 底部的 Home Indicator（横条），尺寸 134 × 5px，圆角 100px，居中显示。

| # | 规则 | 说明 |
|---|------|------|
| 1 | 颜色根据背景选择 | 浅色/白色背景 → **黑色** (#000000)；深色背景 → **白色** (#FFFFFF) |
| 2 | 覆盖在底部按钮区域上 | 当页面有底部按钮区域（Bottom Area）时，Home Indicator 使用绝对定位覆盖在按钮区域的白色背景上，而非单独占一行 |
| 3 | 容器规格 | 外层容器高度 34px，Home Indicator 垂直居中，距底部 8px |

## CellCard 组合规则

| # | 规则 | 说明 |
|---|------|------|
| 1 | CellCard 多模块间距为 **24px** | 当 CellCard 内包含多个子组件（pic / Body / Alert 等）时，模块之间的 gap 为 24px，内边距 padding 保持 16px 不变 |
| 2 | Alert 在 CellCard 内默认使用 **small** 尺寸 | CellCard 中出现 Alert 时，尺寸固定为 small（高度 41~42px，字号 12px） |

## 模块复用优先级规则（警示信息：副标题槽 vs Alert）

出发点是**模块效率：优先复用已有结构槽位，避免动辄新增独立模块**。警示/原因类信息不要默认用 Alert 承载。

| # | 规则 | 说明 |
|---|------|------|
| 1 | 警示/原因类信息优先复用副标题（描述）槽 | 当卡片的副标题/描述槽（如 CardContent 的 `Body medium`、Cardheader 的 `SubtitleText`）为空闲时，将警示信息放入该槽位，并用警示色（`--error` #FF1930）承载语义，**不新增 Alert**。一个组件即可表达，卡片更紧凑 |
| 2 | Alert 仅作"独立模块升级项" | Alert 会额外占行、引入色块与独立边界，本质是把警示**升级为独立模块**。仅当①副标题/描述槽已被有效内容占用（无空间可复用）**且**②该警示重要到需作为独立模块强调时，才使用 Alert |
| 3 | 判断顺序 | 描述槽是否空闲？→ 空 → 复用（普通正文 + 警示色）；→ 已占用且需强调 → 升级为 Alert。**Alert 不是警示的默认样式** |

> 示例：仪容车貌反馈「未通过」卡片中，质检部位为标题、其下的描述槽为空，故将不通过原因放入描述槽并染 `--error` 红色正文，而非另起一条 Alert。

## PRD 对照规则（防止超出范围）

| # | 规则 | 说明 |
|---|------|------|
| 1 | **PRD 没写的内容一律不添加** | 禁止凭设计经验"补全"PRD 未定义的内容，如副文案、列表条目、说明文字等 |
| 2 | 搭建前必须整理文案输入表 | 将 PRD 文案清单逐字段结构化（模块 → 字段 → 文案），有则填，无则标注"不填"，以此作为唯一输入源 |
| 3 | 空状态只显示 PRD 指定文案 | 空状态卡片中禁止自行添加副文案或说明性文字，除非 PRD 明确写明 |

## 组件实例使用规则（防止占位文/多余元素残留）

| # | 规则 | 说明 |
|---|------|------|
| 1 | 每个实例放置后立即执行**组件三问** | ① 有哪些文本属性需要填写？② 有哪些可选项需要关闭（visible=false）？③ 有哪些内容必须通过属性修改而非新建节点？ |
| 2 | **禁止在实例外叠加 TEXT 节点承载组件已有属性** | 如截止时间应写入 CardHeader 的 SubtitleText 属性，而非在卡片容器内额外创建独立 TEXT |
| 3 | CardContent 放置后必须检查并隐藏所有默认可见的多余元素 | 包括：Tag 实例（Component 2）、浮动按钮（Floating Button）、「默认状态」文字按钮等，PRD 未要求的一律 visible=false |
| 4 | CardHeader 副标题（Description/SubtitleText）默认显示占位文 | 放置 CardHeader 后，若 PRD 该模块无副标题字段，必须将 Description 节点 visible=false |
| 5 | CardContent 文案必须在放置后立即填写 | 禁止保留「正文内容示意」「副标题信息内容示意」等组件默认占位文案，放置即填写 |
